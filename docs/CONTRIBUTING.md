# 협업 가이드

팀 과제 시작 후 협업 규칙을 이곳에 정리합니다.

## 브랜치 네이밍 규칙

브랜치명은 **작업 유형/이슈 번호** 형식으로 작성한다.

```bash
작업유형/이슈번호
```

### 예시

```bash
docs/2
feat/3
fix/4
refactor/5
chore/6
```

### 작업 유형

| 유형       | 설명                                 |
| ---------- | ------------------------------------ |
| `feat`     | 새로운 기능 추가                     |
| `fix`      | 버그 수정                            |
| `docs`     | 문서 작성 및 수정                    |
| `refactor` | 기능 변경 없는 코드 구조 개선        |
| `chore`    | 설정, 빌드, 패키지 관리 등 기타 작업 |

### 작성 규칙

- 브랜치명은 소문자로 작성한다.
- 공백은 사용하지 않는다.
- 이슈 번호 앞에는 `#`을 붙이지 않는다.
- `/`를 기준으로 작업 유형과 이슈 번호를 구분한다.
- 하나의 브랜치는 하나의 이슈를 기준으로 생성한다.
